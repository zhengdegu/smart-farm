## 一、部署架构

### 1.1 MVP 阶段：单机 Docker Compose

```
云服务器（4C8G，阿里云/腾讯云）
├── Docker Compose
│   ├── app (Spring Boot 后端)
│   ├── emqx (MQTT Broker)
│   ├── postgres (PostgreSQL + TimescaleDB)
│   ├── redis (缓存 + 会话)
│   ├── prometheus (监控采集)
│   ├── grafana (监控面板)
│   └── nginx (反向代理 + HTTPS)
└── 数据卷
    ├── /data/postgres (数据库)
    ├── /data/emqx (EMQX 持久化)
    └── /data/backup (备份)
```

### 1.2 Docker Compose 配置

```yaml
version: '3.8'

services:
  app:
    image: smart-agri/backend:latest
    ports:
      - "8080:8080"
    environment:
      - SPRING_PROFILES_ACTIVE=prod
      - DB_URL=jdbc:postgresql://postgres:5432/smart_agri
      - DB_USERNAME=${DB_USER}
      - DB_PASSWORD=${DB_PASS}
      - EMQX_HOST=emqx
      - EMQX_PORT=1883
      - JWT_SECRET=${JWT_SECRET}
    depends_on:
      - postgres
      - emqx
      - redis
    restart: always
    deploy:
      resources:
        limits:
          memory: 1536M

  emqx:
    image: emqx/emqx:5.6
    ports:
      - "1883:1883"    # MQTT
      - "8883:8883"    # MQTT TLS
      - "18083:18083"  # Dashboard
    volumes:
      - ./emqx/etc:/opt/emqx/etc
      - emqx_data:/opt/emqx/data
    restart: always
    deploy:
      resources:
        limits:
          memory: 1024M

  postgres:
    image: timescale/timescaledb:latest-pg16
    ports:
      - "5432:5432"
    environment:
      - POSTGRES_DB=smart_agri
      - POSTGRES_USER=${DB_USER}
      - POSTGRES_PASSWORD=${DB_PASS}
    volumes:
      - pg_data:/var/lib/postgresql/data
      - ./sql/init.sql:/docker-entrypoint-initdb.d/init.sql
    restart: always
    deploy:
      resources:
        limits:
          memory: 2048M

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data
    restart: always
    deploy:
      resources:
        limits:
          memory: 256M

  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx/conf.d:/etc/nginx/conf.d
      - ./nginx/certs:/etc/nginx/certs
    depends_on:
      - app
    restart: always

  prometheus:
    image: prom/prometheus:latest
    ports:
      - "9090:9090"
    volumes:
      - ./prometheus/prometheus.yml:/etc/prometheus/prometheus.yml
      - prom_data:/prometheus
    restart: always

  grafana:
    image: grafana/grafana:latest
    ports:
      - "3000:3000"
    volumes:
      - grafana_data:/var/lib/grafana
      - ./grafana/dashboards:/etc/grafana/provisioning/dashboards
    environment:
      - GF_SECURITY_ADMIN_PASSWORD=${GRAFANA_PASS}
    restart: always

volumes:
  pg_data:
  emqx_data:
  redis_data:
  prom_data:
  grafana_data:
```

## 二、监控体系

### 2.1 三层监控

| 层级 | 监控内容 | 工具 | 告警阈值 |
|------|---------|------|---------|
| 基础设施 | CPU/内存/磁盘/网络 | Prometheus + node_exporter | CPU>80%、内存>85%、磁盘>90% |
| 应用层 | JVM指标、API响应时间、错误率 | Spring Boot Actuator + Micrometer | P99>3s、错误率>1% |
| 业务层 | 设备在线率、消息积压、灌溉成功率 | 自定义 Prometheus metrics | 在线率<90%、积压>1000 |

### 2.2 Prometheus 配置

```yaml
# prometheus.yml
global:
  scrape_interval: 15s

scrape_configs:
  - job_name: 'spring-boot'
    metrics_path: '/actuator/prometheus'
    static_configs:
      - targets: ['app:8080']

  - job_name: 'emqx'
    metrics_path: '/api/v5/prometheus/stats'
    static_configs:
      - targets: ['emqx:18083']

  - job_name: 'node'
    static_configs:
      - targets: ['node-exporter:9100']

  - job_name: 'postgres'
    static_configs:
      - targets: ['postgres-exporter:9187']

alerting:
  alertmanagers:
    - static_configs:
        - targets: ['alertmanager:9093']

rule_files:
  - '/etc/prometheus/alert_rules.yml'
```

### 2.3 关键告警规则

```yaml
# alert_rules.yml
groups:
  - name: infrastructure
    rules:
      - alert: HighCPU
        expr: node_cpu_seconds_total{mode="idle"} < 20
        for: 5m
        labels: { severity: warning }
        annotations: { summary: "CPU使用率超过80%" }

      - alert: DiskAlmostFull
        expr: node_filesystem_avail_bytes / node_filesystem_size_bytes < 0.1
        for: 5m
        labels: { severity: critical }
        annotations: { summary: "磁盘剩余空间不足10%" }

  - name: application
    rules:
      - alert: HighErrorRate
        expr: rate(http_server_requests_seconds_count{status=~"5.."}[5m]) > 0.01
        for: 3m
        labels: { severity: warning }

      - alert: EMQXConnectionDrop
        expr: emqx_connections_count < emqx_connections_count offset 5m * 0.5      for: 2m
        labels: { severity: critical }
        annotations: { summary: "EMQX连接数骤降50%以上" }

  - name: business
    rules:
      - alert: DeviceOfflineSpike
        expr: smart_agri_device_offline_count > smart_agri_device_total * 0.3
        for: 5m
        labels: { severity: critical }
        annotations: { summary: "超过30%设备离线" }

      - alert: IrrigationCommandTimeout
        expr: rate(smart_agri_command_timeout_total[10m]) > 0.1
        for: 5m
        labels: { severity: warning }
        annotations: { summary: "灌溉指令超时率偏高" }
```

### 2.4 Grafana 仪表盘

预置 4 个仪表盘：

1. **系统概览** — CPU/内存/磁盘/网络/JVM
2. **EMQX 监控** — 连接数/消息吞吐/Topic统计
3. **业务监控** — 设备在线率/指令成功率/告警统计
4. **数据库监控** — 连接池/查询耗时/表大小/WAL延迟

## 三、日志方案

### 3.1 日志分级

| 日志类型 | 格式 | 保留时间 | 存储位置 |
|---------|------|---------|---------|
| 应用日志 | JSON结构化 | 30天 | 本地文件 + 按天轮转 |
| 设备数据日志 | 写入TimescaleDB | 90天原始/3年聚合 | PostgreSQL |
| 操作审计日志 | JSON结构化 | 1年 | PostgreSQL |
| EMQX日志 | 文本 | 7天 | 本地文件 |

### 3.2 Spring Boot 日志配置

```yaml

    root: INFO
```

## 四、备份策略

### 4.1 备份计划

| 备份项 | 频率 | 保留 | 方式 |
|--------|------|------|------|
| PostgreSQL 全量 | 每天 02:00 | 7天 | pg_dump + gzip |
| PostgreSQL WAL | 实时 | 3天 | WAL归档 |
| EMQX 配置 | 每天 03:00 | 7天 | 文件复制 |
| 应用配置 | 每次变更 | Git | Git仓库 |

### 4.2 备份脚本

```bash
#!/bin/bash
# /opt/scripts/backup.sh — 每天 cron 执行

DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR="/data/backup"
KEEP_DAYS=7

# PostgreSQL 备份
docker exec postgres pg_dump -U $DB_USER smart_agri | gzip > $BACKUP_DIR/pg_$DATE.sql.gz

# EMQX 配置备份
docker cp emqx:/opt/emqx/data/configs $BACKUP_DIR/emqx_$DATE/

# 清理过期备份
find $BACKUP_DIR -mtime +$KEEP_DAYS -delete

# 上传到对象存储（可选）
# aliyun oss cp $BACKUP_DIR/pg_$DATE.sql.gz oss://smart-agri-backup/

# 验证备份完整性
gzip -t $BACKUP_DIR/pg_$DATE.sql.gz && echo "Backup OK: $DATE" || echo "Backup FAILED: $DATE"
```

### 4.3 恢复演练

每月执行一次恢复演练：
1. 在测试环境恢复最新备份
2. 验证数据完整性（设备数、最新数据时间）
3. 记录 RTO（恢复时间目标）：< 1小时
4. 记录 RPO（恢复点目标）：< 24小时（全量）/ < 5分钟（WAL）

## 五、发版流程

### 5.1 发版步骤

```bash
# 1. 构建镜像
docker build -t smart-agri/backend:v1.2.0 .

# 2. 推送到私有仓库
docker push registry.example.com/smart-agri/backend:v1.2.0

# 3. 备份当前版本
docker tag smart-agri/backend:latest smart-agri/backend:rollback

# 4. 更新 docker-compose.yml 中的版本号
sed -i 's/backend:.*/backend:v1.2.0/' docker-compose.yml

# 5. 滚动更新（仅重启 app 服务）
docker compose up -d --no-deps app

# 6. 健康检查（等待30秒）
sleep 30 && curl -f http://localhost:8080/actuator/health || docker compose up -d --no-deps app  # 回滚

# 7. 数据库迁移（Flyway 自动执行）
# Spring Boot 启动时自动执行 Flyway migration
```

### 5.2 回滚方案

```bash
# 快速回滚（< 1分钟）
docker compose stop app
docker tag smart-agri/backend:rollback smart-agri/backend:latest
docker compose up -d app
```

## 六、SLA 保障

### 6.1 可用性目标

| 指标 | 目标 | 计算方式 |
|------|------|---------|
| 系统可用性 | ≥ 99% | 月度（允许宕机 < 7.2小时） |
| RTO（恢复时间） | < 1小时 | 从故障到恢复服务 |
| RPO（数据恢复点） | < 5分钟 | WAL归档保障 |
| 告警响应时间 | < 5分钟 | 从告警触发到运维收到通知 |

### 6.2 故障响应流程

```
告警触发 → Prometheus AlertManager
    ↓
运维微信/短信通知（5分钟内）
    ↓
运维登录服务器排查
    ↓
├── 应用故障 → docker compose restart app
├── 数据库故障 → 检查连接池/磁盘/恢复备份
├── EMQX故障 → docker compose restart emqx
├── 服务器故障 → 联系云厂商 / 切换备机
└── 网络故障 → 网关本地规则兜底，等待恢复
```
