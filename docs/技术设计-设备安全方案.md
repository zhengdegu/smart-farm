## 一、设备身份认证

### 1.1 认证方案：Token + TLS

采用 **EMQX 内置 Token 认证 + TLS 传输加密** 的方案，平衡安全性和实施复杂度。

```
设备注册流程：
1. 设备出厂时烧录唯一 device_id + device_secret
2. 设备首次上线，携带 device_id + device_secret 请求后端 /api/device/auth
3. 后端验证通过后，签发 MQTT 连接 Token（有效期 24h）
4. 设备使用 Token 作为 MQTT password 连接 EMQX
5. Token 过期前自动续签
```

### 1.2 设备注册接口

```java
// POST /api/device/auth
@PostMapping("/device/auth")
public DeviceTokenResponse auth(@RequestBody DeviceAuthRequest req) {
    // 1. 验证 device_id + device_secret
    Device device = deviceService.verify(req.getDeviceId(), req.getDeviceSecret());
    // 2. 签发 JWT Token
    String token = jwtService.generateDeviceToken(device.getId(), Duration.ofHours(24));
    // 3. 返回 Token + MQTT 连接信息
    return new DeviceTokenResponse(token, mqttHost, mqttPort);
}
```

### 1.3 EMQX 认证配置

```yaml
# emqx.conf
authentication = [
  {
    mechanism = password_based
    backend = http
    method = post
    url = "http://backend:8080/api/emqx/auth"
    body {
      username = "${username}"
      password = "${password}"
    }
  }
]
```

## 二、MQTT 安全配置

### 2.1 Topic 命名规范

```
/{tenant_id}/{device_type}/{device_id}/{data_type}

示例：
/t001/sensor/d00001/telemetry     # 传感器上报数据
/t001/valve/d00002/command         # 阀门控制指令
/t001/valve/d00002/command_ack     # 阀门指令确认
/t001/gateway/gw001/status         # 网关状态
```

### 2.2 ACL 权限矩阵

| 角色 | 允许发布 | 允许订阅 |
|------|---------|---------|
| 传感器设备 | `/{tenant}/{type}/{self_id}/telemetry` | `/{tenant}/{type}/{self_id}/config` |
| 控制设备（阀门） | `/{tenant}/{type}/{self_id}/command_ack` | `/{tenant}/{type}/{self_id}/command` |
| 网关 | `/{tenant}/gateway/{self_id}/#` | `/{tenant}/gateway/{self_id}/#` |
| 后端服务 | `/{tenant}/#` | `/{tenant}/#` |

### 2.3 EMQX ACL 配置

```yaml
authorization = [
  {
    type = http
    method = post
    url = "http://backend:8080/api/emqx/acl"
    body {
      username = "${username}"
      topic = "${topic}"
      action = "${action}"
    }
  }

```

## 三、通信加密

### 3.1 TLS 配置

```yaml
# EMQX TLS 监听器
listeners.ssl.default {
  bind = "0.0.0.0:8883"
  ssl_options {
    certfile = "/etc/emqx/certs/server.pem"
    keyfile = "/etc/emqx/certs/server.key"
    cacertfile = "/etc/emqx/certs/ca.pem"
    verify = verify_none  # Token认证，不要求客户端证书
  }
}
```

### 3.2 证书管理

```
证书体系：
├── CA 根证书（自签，有效期 10 年）
├── 服务端证书（EMQX，有效期 1 年，自动续签）
└── 设备端（不使用客户端证书，用 Token 认证降低复杂度）

续签策略：
- 服务端证书到期前 30 天自动续签（Let's Encrypt 或内部 CA）
- 设备 Token 24 小时过期，自动续签
```

## 四、控制指令安全

### 4.1 指令签名

```java
// 指令下发时签名
public String signCommand(CommandPayload payload, String secretKey) {
    String raw = payload.getDeviceId() + payload.getAction() 
               + payload.getTimestamp() + payload.getNonce();
    return HmacUtils.hmacSha256Hex(secretKey, raw);
}

// 指令 payload 示例
{
    "cmd_id": "cmd_20260420_001",
    "device_id": "d00002",
    "action": "OPEN_VALVE",
    "params": { "duration_min": 30 },
    "timestamp": 1713609600,
    "nonce": "a1b2c3d4",
    "sign": "e5f6g7h8..."
}
```

### 4.2 防重放攻击

- 指令携带 `timestamp`，超过 5 分钟的指令拒绝执行
- 指令携带 `nonce`，网关/设备缓存最近 10 分钟的 nonce，重复则拒绝
- 指令携带唯一 `cmd_id`，设备端去重

### 4.3 关键操作二次确认

| 操作 | 是否需要二次确认 |
|------|----------------|
| 开启灌溉（自动触发） | 否 |
| 开启灌溉（手动） | 否 |
| 修改灌溉策略 | 是（小程序弹窗确认） |
| 删除设备 | 是 |
| 修改告警阈值 | 是 |
| 紧急停止所有设备 | 否（紧急操作优先执行） |

## 五、数据安全

### 5.1 敏感数据加密存储

```sql
-- 设备密钥加密存储
CREATE TABLE device (
    id BIGSERIAL PRIMARY KEY,
    device_id VARCHAR(64) UNIQUE NOT NULce_secret_encrypted BYTEA NOT NULL,  -- AES-256 加密
    tenant_id BIGINT NOT NULL,
    status VARCHAR(16) DEFAULT 'ACTIVE',
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 用户密码 bcrypt 哈希
CREATE TABLE sys_user (
    id BIGSERIAL PRIMARY KEY,
    username VARCHAR(64) UNIQUE NOT NULL,
    password_hash VARCHAR(128) NOT NULL,  -- bcrypt
    role VARCHAR(16) NOT NULL
);
```

### 5.2 数据脱敏规则

| 数据类型 | 脱敏规则 | 场景 |

| 设备密钥 | 不可查看 | 任何场景 |
| WiFi 密码 | 不存储 | — |

## 六、安全审计

### 6.1 审计日志

```java
// 所有控制操作记录审计日志
@Aspect
@Component
public class AuditLogAspect {
    @Around("@annotation(Auditable)")
    public Object audit(ProceedingJoinPoint pjp) {
        AuditLog log = new AuditLog();
        log.setOperator(SecurityContext.getCurrentUser());
        log.setAction(getAction(pjp));
        log.setTarget(getTarget(pjp));
        log.setTimestamp(Instant.now());
        log.setResult(pjp.proceed() != null ? "SUCCESS" : "FAILED");
        auditLogRepository.save(log);
        return result;
    }
}
```

### 6.2 异常检测告警

| 异常事件 | 检测规则 | 告警方式 |
|---------|---------|---------|
| 设失败 | 同一 device_id 5分钟内失败 ≥5次 | 封禁设备 + 通知运维 |
| 异常控制指令 | 非工作时间段的手动开阀 | 微信通知管理员确认 |
| 数据异常 | 传感器数据突变超过正常范围 3 倍 | 标记数据可疑 + 告警 |
| 批量设备离线 | 同一网关下 >50% 设备同时离线 | 紧急告警（可能网关故障） |
